# Null/completion closure swing

## Selected portfolio item

This run executes `P2C-NULL-COMPLETION-CLOSURE`, selected from the latest
completed Progress handoff after the finite boundary-adapter swing.

Goal: formalize the physically legitimate middle completion class around the
adapter's completion hook, include omission mutants, and produce a concrete
escape/no-go target without converting ordinary completion into a desired
capability or issuance verdict.

## Closure model

The fixture `tests/physical_completion_closure.py` imports the finite physical
boundary adapter and adds a declared closure over ten middle completion kinds:

- hidden-state;
- boundary;
- seed;
- provenance;
- resource;
- whole-family;
- history;
- access;
- relabeling; and
- gauge completions.

The closure is intentionally finite. A candidate completion may add only a
declared response channel that preserves the normalized resource budget, has
provenance, and does not carry verdict metadata. Relabeling and gauge candidates
are equivalence-only: they can preserve an existing observable effect, but they
cannot introduce a new one.

## Executable laws and controls

Validation command:

```text
python tests/physical_completion_closure.py
```

The script checks:

- every declared middle completion class is represented;
- no legitimate candidate is rejected;
- closure composition is order-insensitive on the generated profile;
- staged closure equals one-shot closure;
- an ordinary trit residual is absorbed by a declared whole-family rival;
- omitting `whole_family` reopens that residual while other single omissions do
  not;
- an undeclared quart response remains an unexplained residual;
- a verdict-carrying completion candidate is rejected; and
- an unbudgeted completion candidate cannot absorb the escape.

## Result

The finite closure absorbs ordinary novelty only when the needed response is in
the declared completion family. It does not make the adapter unrestricted:
undeclared response signatures survive as `UNEXPLAINED_EFFECT_RESIDUAL`, and
the closure rejects both verdict-carrying and resource-frame-changing
shortcuts.

This gives the active lane a sharper falsification target:

> If every physically admissible candidate witness can be generated by this
> declared middle closure under the witness's own normalized resource frame,
> then the witness does not establish capability enlargement at this tier.
> If a response signature remains outside the closure after all admissible
> completions are declared and composed, it becomes a candidate escape requiring
> real physical grounding rather than a conclusion.

## Limits

This is not yet a real physical witness. Exact response signatures are still a
finite stand-in for observable effects. The completion class is executable and
bounded, but its physical admissibility rules must be stress-tested on one
source-grounded system before any capability or finality question can be
meaningfully routed to another repository.
