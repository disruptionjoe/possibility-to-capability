# Cross-frame descent or fork: result

The preregistered result is **`INDEXED_CONSTRUCTION_FORK`**. The unchanged
P2C-W1 evidence survives certified representation relabeling, and the
two-context P2C construction and access-constitutive Frame R agree on this
specimen: at a fixed common interface, the operational profile change is
nonconservative. The one-context whole-family construction reports a different
truth—global containment—because it removes the before/after pair. It is a
material construction fork, not an equivalent redescription and not a
contradiction.

The common object that survives this bounded run is therefore not one universal
English label. It is an indexed primitive profile plus the maps that say which
facts are being preserved.

## What the run actually established

| case | normalized two-context `N` | access-constitutive `R` | whole-family `W` |

All 9 declared model/evidence checks and all 7 failing-direction controls pass.
The only theorem-consequence setup check is reported separately. The source
packet identities, no-authority-transfer flags, and P2C-W1 blob hashes remain
unchanged.

## The coherent story this supports

The hierarchy is more plausible as a typed, indexed explanatory language than
as a single context-free ladder of substances.

1. A family may contain a possible realization without every starting context
   being able to realize it.
2. A fixed interface can expose unequal operational task profiles across two
   contexts.
3. A different choice of system, interface, starting point, resources, or task
   can change the capability object itself. That is a construction fork unless
   a faithful transport has been supplied.
4. Equivalent representations must preserve the full primitive profile and
   each frame's output. A relabel that changes a verdict is a genuine F5
   failure; no such failure occurred here.
5. Global containment and local operational realizability are distinct
   predicates. Preserving both is more informative than forcing either one to
   erase the other.

The strongest candidate joint object is a restriction diagram:

```text
                    whole family W
             target realization is contained
                    /             \
          restrict to N         restrict to S
              /                     \
      task profile absent       task profile present
```

This diagram can also be read by Frame R when one interface is frozen: the
lower fibers have different verified task profiles. What is not yet proved is
that the restriction maps exist with all task, resource, interface, record,
and cost data preserved.

## Important negative result

Frame R is not an opposing verdict on P2C-W1. It challenges any
access-independent or frame-free interpretation of capability, and it rejects
privileging an interface without justification. But once one common interface
is stipulated, its own rules classify the `(1,1,1)` versus `(0,0,0)` profile as
a nonconservative capability change. The sharp disagreement appears on the
access-only control: P2C calls it access; R calls it conservative capability
extension. The primitive facts remain identical.

That means this run does not choose a winner between P2C and R. It locates their
difference in the ownership boundary of the capability object.

## Specification incident retained

The first execution exposed an ambiguity in the calibration loader. The
machine-readable relabel case listed a partial overlay, but a generic baseline
fill treated omitted whole-family containment as false. That silently changed
the object under an allegedly equivalent relabel and produced a false
`F5_EQUIVALENCE_FAILURE`.

The repair makes the relabel inherit the operational P2C-W1 profile before
applying its declared overlay, exactly as preregistered P4 requires. No frame
rule, outcome rule, prediction, or source input changed. This incident is a
useful process result: equivalence fixtures must declare their inheritance base
or fully materialize every primitive field.

## Strongest live attacks

The collapse-side audit leaves five serious attacks open:

1. **Interface-constitutive control:** a latch/verifier added only through the
   interface can create a task profile without changing the conductor.
2. **Preparation-inclusive reachability:** if `COOL` remains an admissible
   evaluation action with adequate budget, the N starting fiber may reach S.
3. **Resource-profile reduction:** an active finite-horizon controller may turn
   a binary task difference into a cost–horizon–error Pareto difference.
4. **Target-loaded task semantics:** winding and zero maintenance may encode the
   winning carrier into the task vocabulary; C1 remains live.
5. **Augmented-dynamics reduction:** after system, interface, records,
   resources, controller, and verifier are internalized, capability may be only
   a useful reachability projection rather than an independently forced type.

These attacks do not erase the observed profile difference. They test what the
difference means and which explanatory level owns it.

## Next decisive construction

Build a finite raw-transition restriction model that freezes:

- a common carrier span and starting-context embeddings;
- preparation versus evaluation actions;
- one task-and-verification interface;
- vector resources, horizon, and error tolerance;
- full reachable-policy closure; and
- pullbacks from the whole-family envelope to each starting fiber.

Then run two conclusions side by side: the restriction-diagram theorem and the
augmented-dynamics/bisimulation reduction. If unequal fibers survive a faithful
pullback and task-preserving quotient, the scoped capability distinction
strengthens. If they equalize, collapse is earned. If the maps cannot be
defined without changing system identity or task semantics, the honest result
remains an indexed construction fork.
