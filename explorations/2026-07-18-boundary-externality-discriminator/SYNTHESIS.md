# Boundary externality and signal discriminator

## Result

The 2026-07-18 boundary-externality intake is now converted into a
carrier-neutral executable discriminator. The discriminator does not decide
that anything is outside the universe, that a source repository has a native
boundary current, that GU has a firewall, or that any finality or capability
claim moved. It does one bounded P2C job: before future packet combinations
use "outside" or "signal" language, candidate classes must declare their
interface, condition-versus-signal status, transfer, timing, closure, resource
and access frame, initiation and maintenance role, completion absorption,
after-fact encoding risk, and nearest observable null distinction.

The companion fixture is:

```text
tests/boundary_externality_signal_discriminator.py
```

It classifies or abstains on the preserved intake classes:

- apparent boundary or representation;
- static global datum;
- initial or boundary condition;
- two-ended, final-boundary, or fixed-point condition;
- observer or agent selection, split into epistemic readout and physical
  intervention;
- autonomous bulk-boundary feedback;
- edge, defect, membrane, or anomaly-inflow degrees of freedom;
- enlarged-system resource reservoir;
- stochastic environment or noise;
- native source current or law-like update; and
- meta-system or ensemble selection.

## Discriminator rule

The classifier is deliberately conservative:

1. If a candidate depends on a source-native current, law update, or physical
   boundary construction, it returns `ABSTAIN_SOURCE_SOVEREIGNTY` until a
   frozen source packet exists.
2. If the candidate is an after-fact selector or outcome encoding, it returns
   `NOT_A_DELIVERED_SIGNAL`.
3. If a fixed condition is absorbed by a completed family, it returns
   `FIXED_FAMILY_COMPLETION`, not capability.
4. If a two-ended or fixed-point condition is suggested, it returns
   `ABSTAIN_FACTOR_SEARCH_REQUIRED`; finality is not licensed by fixed-point
   vocabulary alone.
5. If only observer projection changes, it returns `ACCESS_CHANGE_ONLY`.
6. If the effect depends on added energy or resources under an unmatched
   frame, it returns `RESOURCE_FRAME_CHANGED`.
7. If the boundary relation is stochastic or underdeclared, it abstains until
   timing, record, and transfer are pinned.
8. If autonomous feedback closes but produces no matched task delta, it
   remains maintenance closure, not capability.
9. Only the positive calibration case reaches
   `CAPABILITY_CHANGE_CANDIDATE_NO_FINALITY`, and only because it declares a
   closed state-dependent channel, matched resource and access frame, and a
   task delta under that frame.

This is intentionally not a verdict machine. It blocks fluent relabeling of
representation, access, resources, or after-fact history selection as
capability.
